기능 설명 [김건희]

[로그인 창]
아이디와 비밀번호 일치시에 해당되는 계정의 데이터를 가져와 로그인이 완료된 상태의 메인페이지로 이동한다. 

- 로그인 창에서 비밀번호나 아이디 입력이 없을 경우 로그인 과정으로 넘어가지 않음
- 회원 가입 창 링크 
- 아이디 찾기 창 링크
- 비밀번호 찾기 창 링크 

[아이디 찾기]
이름과 휴대전화를 입력값으로 받아서 거기에 대응하는 정보를 찾아서 아이디를 가르쳐 준다.

- 사용자가 아이디 찾기 이후 비밀번호 찾기로 넘어가거나 로그인 페이지로 돌아간다. 

[비밀번호 찿기]
아이디와 이름과 휴대전화를 입력값으로 받아서 거기에 대응하는 정보를 찾아서 비밀번호를 변경할 수 있도록 한다.

- 보안의 원칙에 의거하여 비밀번호의 길이가 8자리가 되지 않는 경우에 변경되지 않음

[회원 가입]
아이디, 비밀번호, 이름, 성별, 이메일, 소속 지점의 데이터를 입력받아서 회원 가입을 진행한다.

- 각 데이터의 하나의 값만 없더라도 회원 가입이 진행되지 않는다.
- 아이디의 길이가 올바르지 않거나, 아이디 체크 기능을 이용하여 중복되는 아이디일 경우에 해당 아이디로 생성 불가
- 비밀번호와 비밀번호 확인이 일치하지 않을 경우, 비밀번호의 길이가 보안의 원칙에 의거한 최소 8자리가 되지 않을 경우에 회원 가입이 진행되지 않는다.
- 아이디 중복확인 이후 변경이 된 경우에는 알림창으로 다시 중복확인 필요하다고 알려 준다.
- 이메일 형식이 알맞지 않은 경우에 회원 가입이 진행되지 않는다. 

[예약 페이지 이동] 
예약 페이지 이동시에 로그인이 되어 있지 않은 상태일 경우 알림창이 뜨면서 예약 페이지는 로그인 상태에서 진행하라고 알림 

[관리자 기능 -회원 복구 기능-] 
이미 탈퇴한 회원이 회원 복구를 원할시에는 회원이 데이터를 복구할 수 있는 구현을 구현 

[로그 아웃]
로그아웃시에 세션에 남아 있는 값을 날리고 로그인이 되어 있지 않은 메인 페이지 화면으로 이동한다.

[게시판]
- 게시판의 로그인 여부에 따른 HTML 출력 내용 차이를 두도록하는 로직을 구현 
- 게시판 글 입력, 수정, 삭제시에 관련된 기능 추가 구현 및 데이터 파라미터 설정

▶ TOP : 페이지 내에서 상단으로 이동됩니다.
